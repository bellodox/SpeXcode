#[Memory Bank: Active] Project Brief

This repository is an extension-only SpeXcode monorepo focused on the native VS Code extension product graph. Retained active surfaces remain [`src/`](src), [`webview-ui/`](webview-ui), and required internal packages under [`packages/`](packages); removed product surfaces remain [`apps/`](apps), [`jetbrains/`](jetbrains), [`packages/evals/`](packages/evals), and [`packages/agent-runtime/`](packages/agent-runtime).

The latest completed user-facing docs/UI pass finished the cosmetic rebrand from Kilo Code to SpeXcode and updated live repository links to `https://github.com/bellodox/SpeXcode`, while intentionally retaining compatibility identifiers such as extension IDs, publishers, command/config keys, storage keys, package names, and `kilocode`-named implementation paths. A subsequent implementation pass changed the active config directory convention from [`.kilocode`](.kilocode) to [`.spexcode`](.spexcode) with [`.kilocode`](.kilocode) preserved as a fallback, and updated the extension manifest identity to `spex.spex-code`. This memory bank remains local-only and noncanonical. Carbon Memory in this repository combines this volatile memory bank, the durable maintainer wiki under [`docs/maintainer-wiki/`](docs/maintainer-wiki/), and indexed repository context from entrypoint documents such as [`AGENTS.md`](AGENTS.md) and [`README.md`](README.md); durable facts belong in [`docs/maintainer-wiki/index.md`](docs/maintainer-wiki/index.md).
